知识库部分
# 核心目的
    独立化知识库模块，用户可以通过KB_factory来无缝调用多重格式的知识库。
    开发者也可以通过继承base模块中的BASE_KB类来扩展知识库的功能。

# 整体框架
1. base 基础模块 
    用于设置知识库的基类
    核心类：
        —— BaseKBConfig: 定义知识库的配置模型
        —— BASE_KB: 定义一个抽象的基类，用于定义知识库的接口
        —— KB_factory: 用于定义一个知识库工厂，用于加载多格式的知识库。
        —— KB_manager： 用于管理知识库，用于加载知识库，并返回知识库的实例
